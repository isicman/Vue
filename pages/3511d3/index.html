# CloudKitty

# 前言

什么是CloudKitty？ CloudKitty是OpenStack等的评级即服务项目。该项目旨在成为云的退款和评级的通用解决方案。从历史上看，它只能在OpenStack上下文中运行它，但现在可以在独立模式下运行CloudKitty。

CloudKitty允许进行基于指标的评级：它轮询终端节点以检索有关特定指标的度量值和元数据，将评级规则应用于收集的数据，并将评级数据推送到其存储后端。

CloudKitty是高度模块化的，这使得添加新功能变得容易。

# 架构

CloudKitty可以分为四大部分：

  • Data retrieval (API) 数据检索
  • Data collection (cloudkitty-processor) 数据收集
  • Data rating 数据评级
  • Data storage 数据存储

这些部分由两个进程处理：和 。数据检索部分由进程处理，其他部分由 处理。cloudkitty-apicloudkitty-processorcloudkitty-apicloudkitty-processor

# 评分

CloudKitty 附带三个评级模块：

  • noop：用于测试目的的评级模块（仅启用）。

  • hashmap：对应于常见 CloudKitty 用例的默认评级模块（默认禁用）。

  • pyscripts：自定义评级模块，允许您添加自己的 python 脚本（默认禁用）。

您可以独立启用或禁用每个模块，并随意优先考虑一个模块。

  • Enabled状态由布尔值（True或False）表示。
  • Priority由一个整数值表示。

# Hashmap评级模块

HashMap 由不同的资源和组组成。

# 组

组是一种对映射计算进行分组的方法。例如，您可能希望应用一组规则来评估实例正常运行时间，并应用另一组规则来阻止存储卷。您不希望将两者联系起来，因此您将为每个计算创建一个组。

# 服务

服务是一种将规则映射到收集的数据类型的方法。必须为您要评价的每种指标类型创建一个 hashmap 服务。如果指标有alt_name，则 hashmap 服务的名称必须与 alt_name. 如果未alt_name提供，请使用指标的名称。

默认配置示例：

metrics:
  cpu:
    unit: instance
    alt_name: instance
  # [...]

  image.size:
    unit: MiB
  # [...]

# 字段

字段是指资源的元数据字段。例如，在一个实例对象上（在instance服务中），您可以使用风格来定义特定的规则。

配置中指定的每个groupby和metadata属性都可以用于一个字段：

metrics:
  cpu:
    unit: instance
    alt_name: instance
    groupby:
      - id
      - project_id
    metadata:
      - flavor_id
    # [...]

  volume.size:
    unit: GiB
    groupby:
      - id
      - project_id
    metadata:
      - volume_type
    # [...]

# 映射

映射是最终对象，它触发计算，例如实例上的特定风味值。

有两种映射：字段映射和服务映射。

字段映射¶ (opens new window)

字段映射用于匹配资源的属性/元数据。例如，如果您要对三种卷类型应用不同的评级规则，则必须按以下方式进行：

  1. 创建一个匹配名称或alt_name卷度量的哈希图服务（volume.size使用默认 gnocchi）。
  2. 在该服务中，使用卷类型元数据的名称创建一个字段（volume_type使用默认 gnocchi）。
  3. 在该字段中，为每个可能的元数据值创建一个映射volume_type 。例子：
    • SSD_gold: 0.03
    • SSD_silver: 0.02
    • HDD_bronze: 0.01

体积指标的每个元素现在都将基于其volume_type 元数据。10GiBSSD_gold卷将被评为每个收集周期0.3，1GiB 卷将被评为 0.01，0.5GiB 将被HDD_bronze评为SSD_silver0.01……

# 服务映射

服务映射不与字段相关联，而是直接与服务相关联。如果直接在服务上创建映射volume.size，则将根据此映射对每个卷进行评级，没有基于元数据的区别。

# 统一和费率

映射可以有两种类型：flat或rate。平面映射只是简单地添加到给定项目的总数中，而比率乘以总数。

# 范围

可以将映射绑定到特定的范围/tenant_id。

# 阈值

阈值条目仅用于在特定级别之后应用评级规则。除此之外，它的工作方式与映射相同。

至于映射，阈值可以与特定范围/项目相关联。
