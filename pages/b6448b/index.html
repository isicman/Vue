    # Python SDK调用API

    # 如何使用SDK

    # SDK的编写

    1. 通过Python SDK直接调用OpenStack API接口的资源，对其使用以及自动化运维。

    作为 OpenStack 用户或管理员，您常常需要编写脚本来自动化常见任务。除了 REST 和命令行接口之外，OpenStack 还公开了原生的 Python API 绑定。了解如何使用这些 Python 绑定简化编写OpenStack 自动化脚本的过程。

    1. OpenStack 是一个越来越流行的、用于部署基础架构即服务 (IaaS) 云的开源解决方案。OpenStack 附带了一个仪表板 Web 应用程序，非常适合执行手动任务，比如启动单个虚拟机 (VM) 实例，但是，如果希望自动化基于云的任务，则需要编写可操作 OpenStack 的脚本。

    2. 许多用户直接为 OpenStack 具象状态传输 (REST) 应用编程接口 (API) 编写自动化脚本，或者编写调用命令行工具（比如 keystone 或 nova ）的 shell 脚本。但 Python 中有一种编写 OpenStack 自动化脚本的更好方法。所有 OpenStack 服务都公开了原生 Python API，以公开与命令行工具相同的特性集。不幸的是，描述如何使用这些 API 的文档很少。

    # SDK的优点

    • 方便高效率性
    • 通过调用OpenStack的sdk来使用对应的模块
    • 针对每一个接口都有对应的函数，包括对应的函数使用说明

    # OpenStack Neutron

    Neutron 是一个在OpenStack计算服务管理的设备之间提供虚拟网络服务的项目。

    v2.0 API 代表了Quantum v1.1 API与 (opens new window)Melange API (opens new window)中一些最重要的 IPAM 功能的组合。这些 IPAM 功能侧重于能够将网络设备（例如，默认网关、dns 服务器）所需的 IP 地址块和其他网络配置与 Neutron 网络相关联，然后能够从这样的块中分配 IP 地址并将其与通过 Neutron 端口连接到网络的设备相关联。

    v2.0 API 通过引入一个称为子网的新实体来做到这一点。子网可以表示 IPv4 或 IPv6 地址块与先前创建的 Quantum 网络的绑定，并且每个 Neutron 网络通常具有一个或多个子网。在网络上创建端口时，默认情况下，它将从每个 IP 版本的指定子网中分配一个可用的固定 IP 地址。当端口被销毁时，分配的地址将返回到子网上的可用 IP 池中。Neutron API 的用户可以从块中选择一个特定的 IP 地址，或者让 Neutron 选择第一个可用的 IP 地址。

    Neutron v2 API 管理三种实体：

    • Network，代表隔离的虚拟二层域；网络也可以被视为虚拟（或逻辑）交换机；
    • Subnet，表示 IPv4 或 IPv6 地址块，从中选择要分配给给定网络上的 VM 的 IP。
    • Port，表示给定网络上的虚拟（或逻辑）交换机端口。

    # 1.创建网络

    通过调用OpenStack SDK对neutron服务进行运维操作，调用对应的sdk来创建一个网络，使用的参数如下。

    • 创建一块网卡名称为：openstacksdk-example-project-network
    • 创建一个子网名称为：openstacksdk-example-project-subnet
    • 子网的使用ipv4的版本
    • 子网网段是： 10.0.2.0/24
    • 子网的网关是： 10.0.2.1
    import openstack

def create_connection(auth_url, region, project_name, username, password,
                      user_domain, project_domain):
    return openstack.connect(
        auth_url=auth_url,
        project_name=project_name,
        username=username,
        password=password,
        region_name=region,
        user_domain_name=user_domain,
        project_domain_name=project_domain,
        app_name='examples',
        app_version='1.0',
    )

conn = create_connection("http://controller:5000/v3/","RegionOne","admin","admin","000000","demo","demo")

def create_network(conn):
    print("Create Network:")

    example_network = conn.network.create_network(
        name='openstacksdk-example-project-network')

    print(example_network)

    example_subnet = conn.network.create_subnet(
        name='openstacksdk-example-project-subnet',
        network_id=example_network.id,
        ip_version='4',
        cidr='10.0.2.0/24',
        gateway_ip='10.0.2.1')

    print(example_subnet)
create_network(conn)

    # 2.创建规则

    通过调用OpenStack SDK对neutron服务进行运维操作，调用对应的sdk来创建一个安全组规则，使用的参数如下。

    1.创建一个开放端口的规则：

    • 需要创建的安全组名称为：openstacksdk-example-security-group
    • 允许入口访问流量的默认地址为：全部放行
    • 开放端口： 443
    • 端口协议： TCP
    • 网络的类型是： IPv4

    2.创建一个允许所有ping的规则：

    • 需要创建的安全组名称为：openstacksdk-example-security-group2
    • 允许入口访问流量的默认地址为：全部放行
    • 端口协议： ICMP
    • 网络的类型是： IPv4
    import openstack

def create_connection(auth_url, region, project_name, username, password,
                      user_domain, project_domain):
    return openstack.connect(
        auth_url=auth_url,
        project_name=project_name,
        username=username,
        password=password,
        region_name=region,
        user_domain_name=user_domain,
        project_domain_name=project_domain,
        app_name='examples',
        app_version='1.0',
    )

conn = create_connection("http://controller:5000/v3","RegionOne","admin","admin","000000","demo","demo")

def open_port(conn):
    print("Open a port:")

    example_sec_group = conn.network.create_security_group(
        name='openstacksdk-example-security-group')

    print(example_sec_group)

    example_rule = conn.network.create_security_group_rule(
        security_group_id=example_sec_group.id,
        direction='ingress',
        remote_ip_prefix='0.0.0.0/0',
        protocol='tcp',
        port_range_max='443',
        port_range_min='443',
        ethertype='IPv4')

    print(example_rule)


def allow_ping(conn):
    print("Allow pings:")

    example_sec_group = conn.network.create_security_group(
        name='openstacksdk-example-security-group2')

    print(example_sec_group)

    example_rule = conn.network.create_security_group_rule(
        security_group_id=example_sec_group.id,
        direction='ingress',
        remote_ip_prefix='0.0.0.0/0',
        protocol='icmp',
        port_range_max=None,
        port_range_min=None,
        ethertype='IPv4')

    print(example_rule)
open_port(conn)
allow_ping(conn)

    # OpenStack Nova

    OpenStack Nova Service负责管理整个Compute Service的整个生命周期。 对Nova API的管理依然需要使用对应的凭证访问openstack，基于凭证对整个Compute Service生命周期实现自动化运维。

    # 1.创建Flavor

    通过调用OpenStack SDK对nova服务进行运维操作，调用对应的sdk来创建一个实例类型，使用的参数如下。

    • 实例类型的名称为： vm
    • 实例类型的磁盘大小为：10G
    • 实例类型的vcpus为： 2个
    • 实例类型的内存为： 2048
    • 实例类型的id为：1234
    import openstack

def create_connection(auth_url, region, project_name, username, password,
                      user_domain, project_domain):
    return openstack.connect(
        auth_url=auth_url,
        project_name=project_name,
        username=username,
        password=password,
        region_name=region,
        user_domain_name=user_domain,
        project_domain_name=project_domain,
        app_name='examples',
        app_version='1.0',
    )
conn = create_connection("http://controller:5000/v3/","RegionOne","admin","admin","000000","demo","demo")

def create_flavor(conn):
    flavor_name = conn.create_flavor(
                    name="vm",
                    disk="10",
                    vcpus="2",
                    ram="2048",
                    flavorid="1234"
    )
    print(flavor_name)

    # 2.创建云主机

    通过调用OpenStack SDK对nova服务进行运维操作，调用对应的sdk来创建一个虚拟机，使用的参数如下。

    • 创建云主机使用名称： vms
    • 镜像来源使用： 通过搜寻cirros的id
    • 实例类型来源使用： 通过搜寻flavor的id
    • 网络使来源使用：搜寻Network名称的网卡id
    import openstack

def create_connection(auth_url, region, project_name, username, password,
                      user_domain, project_domain):
    return openstack.connect(
        auth_url=auth_url,
        project_name=project_name,
        username=username,
        password=password,
        region_name=region,
        user_domain_name=user_domain,
        project_domain_name=project_domain,
        app_name='examples',
        app_version='1.0',
    )
conn = create_connection("http://controller:5000/v3","RegionOne","admin","admin","000000","demo","demo")

def create_server(conn):
    print("Create server:")
    image = conn.get_image_id(image_name="cirros")
    flavor = conn.get_flavor(name_or_id="vm")
    network = conn.get_network(name_or_id="Network")

    server_name = conn.create_server(
        name="vms",
        image=image,
        flavor=flavor,
        network=network
    )
    print(server_name)
create_server(conn)

    # OpenStack Cinder

    Cinder在openstack中提供对卷从创建到删除整个生命周期的管理，从虚拟机 (opens new window)实例的角度来看，挂载的每一个卷都是一块硬盘。openstack提供块存储服务的是cinder，具体功能如下：

    • 提供api使用户能够查询和管理卷、卷快照以及卷类型
    • 提供scheuler调度卷创建请求，合理优化存储资源的分配
    • 通过driver架构支持多种后端存储方式

    # 1.创建硬盘

    通过调用OpenStack SDK对Cinder服务进行运维操作，调用对应的sdk来创建一块硬盘，使用的参数如下。

    • 硬盘的名称为： volume
    • 硬盘的大小为： 10G
    import openstack

def create_connection(auth_url, region, project_name, username, password,

conn = create_connection("http://controller:5000/v3","RegionOne","admin","admin","000000","demo","demo")
