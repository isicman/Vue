# Solidity基础

# 1.Solidity的了解

# 1.1 Solidity语言简介

Solidity是一种面向对象的编程语言 (opens new window)，用于在各种区块链 (opens new window)平台上实现智能合约 (opens new window)[2] (opens new window)[3] (opens new window) ，最著名的是以太坊 (opens new window)。[4] (opens new window)它由 Christian Reitwiessner、Alex Beregszaszi 和几位前以太坊 (opens new window)核心贡献者开发。[5] (opens new window) Solidity中的程序在以太坊虚拟机 (opens new window)上运行。

Solidity虽然是面向智能合约的编程语言。也即Solidity发明出来的目的就是为了实现智能合约。但是请注意，智能合约的实现手段不止是Solidity，其他的语言比如Go, JS等同样可以用来实现智能合约。只不过Solidity专门用来做这件事情，是不是就比其他的语言更好用，需要慢慢发现。

# 1.2 历史

Solidity 由Gavin Wood (opens new window)于 2014 年 8 月提出；[6] (opens new window) [需要非主要来源 (opens new window)]该语言后来由 Christian Reitwiessner 领导的以太坊项目的 Solidity 团队开发。

Solidity 是以太坊 (opens new window)以及其他私有区块链（例如面向企业的Hyperledger (opens new window) Fabric区块链）上的主要语言。SWIFT (opens new window)使用在 Hyperledger Fabric 上运行的 Solidity 部署了概念验证。

# 1.3 说明

Solidity 是一种静态类型的编程语言 (opens new window)，旨在开发在以太坊虚拟机 (opens new window)(EVM) 上运行的智能合约。[9] (opens new window)

Solidity 使用类似ECMAScript (opens new window)的语法，这让现有的 Web 开发人员很熟悉；[需要引用 (opens new window)]但是与 ECMAScript 不同的是，它具有静态类型和可变 (opens new window)返回类型。Solidity 在一些重要方面不同于其他针对 EVM 的语言，例如 Serpent 和 Mutan。它支持合约的复杂成员变量，包括任意层次的映射和结构 (opens new window)。Solidity 合约支持继承 (opens new window)，包括带有C3 线性化的 (opens new window)多重继承 (opens new window)。Solidity 引入了应用程序二进制接口 (opens new window)(ABI)，可促进多种类型安全 (opens new window)单个合约中的函数（这后来也得到了 Serpent 的支持）。Solidity 提案还包括“自然语言规范”，这是一个文档系统，用于指定以用户为中心的方法调用分支的描述。

# 1.6 Solidity 的局限性

与可以调试的传统编程语言中的程序不同，Solidity 合约中的错误无法编辑或修复；交易不能撤销。Solidity 遵循“代码就是法律”的口号，这意味着任何智能合约在生效时都必须经过完美编码。

有一些黑客案例，例如前面提到的 2016 年 DAO 黑客事件，其中6000 万美元 (opens new window)被盗，以及 2021 年黑客事件导致以太坊系统分叉。

为了防止技术失误和失误，美国最大的加密货币交易所Coinbase推出了一款名为 Solidify 的新工具。 (opens new window)该工具是一个人工智能审计系统，用于检测和分类智能合约风险。

# 2.Solidity基础实战

# 2.1 第一个案例

//声明版本号
pragma solidity ^0.4.16;
// pragma solidity >=0.7.0 <0.9.0

//合约 有点类似于java中的class
contract HelloWorld{
    //合约属性变量
    string myName = "HelloWorld";
    //合约中方法 注意语法顺序 其中此处view 代表方法只读 不会消耗gas
    function getName() public view returns(string){
        return myName;
    }
    //可以修改属性变量的值 消耗gas
    function changeName(string _newName) public{
        myName = _newName;
    }
    // pure:不能读取也不能改变状态变量
    function pureName(string _name) public pure returns(string){
        return _name;
    }
}

修饰符详解：

用constant、view、pure修饰function分别表示：

  • constant:只能读取不可改变状态变量(就是contract中定义的变量）
  • view:只能读取不可改变状态变量,和constant一样
  • pure:不能读取也不能改变状态变量

# 2.2.4 Import关键字

import关键字作用： 导入其他源文件

Solidity所支持的导入语句import，语法和JavaScript（从 ES6起）非常类似。

主要的import的使用如下：

  • import "filename";

    从"filename"中导入所有的全局符号到当前的全局作用域中

  • import * as symbolName from "filename";

    创建一个新的全局符号 symbolName,其成员均来自"filename" 中全局符号

  • import "filename" as symbolName;

    这条语句等同于import * as symbolName from "filename";

  • import {symbol1 as alias, symbol2} from "filename";

    创建新的全局符号alias和symbol2，分别从"filename"引用symbol1和symbol2

例子：

编写DemoB.sol的合约。

pragma solidity >=0.7.0 <0.9.0;

contract DemoB {
    // 定义一个字符串变量为Lisi
    string secondName = "Lisi";

    // 定义一个函数用来获取变量
    function getSecondName() public view returns (string memory){
        return secondName;
    }
}

# 2.2.5 EVM虚拟机

一个类似JVM的东西，叫做EVM。

以太坊虚拟机上运行的是合约的字节码形式，需要在部署之前先对合约进行编译。

以太坊虚拟机是一个被沙箱封装起来，完全隔离的运行环境。运行在EVM内部的代码不能接触到网络，文件系统或者其他的进程，即使是智能合约与其他的智能合约也只能进行有限的交互。
