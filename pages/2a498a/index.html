# Barbican

# Barbican简介

什么是Barbican？

Barbican 是 OpenStack 密钥管理器服务。它提供秘密数据的安全存储、配置和管理。这包括密钥材料，例如对称密钥、非对称密钥、证书和原始二进制数据。

Barbican的核心是加解密以及密钥的存储。基本思想是初始化时生成一个根密钥，安全的存储在硬件HSM中，无法读出，只能进行加解密操作。所有的应用密钥在存储在数据库中之前，都是经过根密钥加密的，因此不怕数据库被拖库。使用密钥时，从数据库读出密文，用根密钥解密后获取明文，再使用。

密钥存储、生成、加解密这部分，barbican充分利用的openstack (opens new window)的插件思想，进行抽象化设计，可以支持不同的软件、硬件密钥管理方案，如PKCS11硬件、KMIP硬件等。

# 功能

Barbican能够存储的“秘密”包括：

  • 密钥 (opens new window)。包括对称算法中的密钥、非对称算法中的私钥（包括公钥）、数字签名的密钥等。其底层通过plugin设计支持不同的软、硬件（HSM）。
  • 证书。通过plugin的方式提供证书的颁发、验证等功能。此功能将在Pike版本废弃。
  • 二进制数据。

# 抽象概念解释

Secrets

Secrets是需要存储的密钥 (opens new window)，可能的形式如：私钥、证书、密码等。

ACL

除了基于角色的权限控制外，ACL可以提供基于用户访问列表的权限控制。即可以指定哪些用户对该密钥有访问权限 (opens new window)。

Orders

order允许用户请求barbican自己产生secret。并且支持异步生成secret。当希望请求生成多个公私钥对时，非常有用。一个order生成一个对应的secret，order可以异步生成。

order支持如下两种类型：

  • symmetric keys 对称密钥
  • asymmetric keys 非对称密钥

Containers

Containers是一个容器，用于存储多个secret引用。当不得不跟踪、访问上百个secret时，container (opens new window)非常有用。

barbican支持3种类型的container：

  • Generic
  • Certificate
  • RSA

每种类型的container应该保存什么类型的secret，有明确的限制。

Generic Containers

用于存储任意类型的secret。对于secret的类型和存储数量没有任何限制。

# 更新和删除密钥

您无法更改密钥有效负载（除了删除密钥），但如果您在未指定有效负载的情况下创建了密钥，update则可以稍后使用函数添加有效负载。下面是一个例子。

[root@controller ~]# openstack secret update http://controller:9311/v1/secrets/25879b82-480a-4076-8cad-0edcb7b4f329 'TestPlayload-update'

[root@controller ~]# openstack  secret delete http://controller:9311/v1/secrets/25879b82-480a-4076-8cad-0edcb7b4f329

# 对称密钥生成

对称密钥适用于特定任务，例如 nova 磁盘加密和 swift 对象加密。

--mode：可以将生成的密钥设置为使用特定模式，例如、ctr或cbc有关详细信息，请参阅NIST SP 800-38A。

# 备份和恢复密钥

备份和恢复加密密钥的过程取决于后端的类型。

备份和恢复一个简单的加密后端

一个简单的加密后端需要备份两个不同的组件（KEK 和数据库）。建议定期测试备份和还原过程。

1.KEK 备份和恢复

一个简单的加密后端barbican.conf需要备份包含主 KEK 的文件。此文件应备份到更安全的位置。实际数据以加密方式存储在 Barbican 数据库中，如下一节所述。

  • 要从备份中恢复密钥，您需要将恢复的密钥复制 到barbican.conf现有密钥中。barbican.conf

2.后端数据库备份与恢复

此过程描述如何为简单的加密后端备份和恢复 barbican 数据库。这将生成一个密钥并将密钥上传到 barbican。然后备份 barbican 数据库并删除您创建的秘密。然后恢复数据库并确保您之前创建的密钥已恢复。

# 3.恢复密钥

要从备份中恢复 barbican 数据库，请使用 barbican 权限登录 Controller 节点并恢复 barbican 数据库。要从备份中恢复 KEK，请使用barbican.conf备份文件覆盖该文件。

先决条件

  • OpenStack 密钥管理器已安装并正在运行
  • 您有barbican.conf文件和 barbican 数据库 的现有备份

恢复数据库

登录到controller-0节点并检查您barbican在控制器上是否拥有授予barbican用户访问权限以恢复数据库的数据库：
